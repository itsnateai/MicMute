## Project Summary

MicMute is a Windows system tray utility that provides a global hotkey to toggle microphone mute/unmute, with visual tray icon feedback, audio cues, on-screen display, and a full settings GUI. Built in AutoHotkey v2, it uses COM automation (Windows Core Audio APIs) for direct mic control and supports push-to-talk, deafen mode, device selection, and custom sounds/icons.

## Key Features

- **Global hotkey** (Win+Shift+A default) — mute/unmute from any app
- **System tray indicator** — green (active) / red (muted) with flash animation
- **Push-to-talk mode** — hold to unmute, release to mute (30s safety timeout)
- **Deafen mode** — separate hotkey mutes mic + speakers simultaneously
- **On-screen display** — floating MUTED/ACTIVE overlay above taskbar
- **Settings GUI** — full graphical settings for all options (behavior, OSD, hotkeys, custom files)
- **Audio device selector** — pick specific mic from tray submenu, persists in INI
- **Custom sounds and icons** — user-configurable .wav and .ico files
- **Mute lock** — prevents external apps from changing mute state
- **Run at startup** — one-click toggle to create/remove Windows startup shortcut
- **Mute on lock** — auto-mute when PC locks (Win+L)
- **Unmute on exit** — auto-restores mic on script exit
- **Help Window** — in-app help covering all features and troubleshooting
- **Embedded icons** — .ico files embedded as PE resources, no external files needed
- **ToolTip notifications** — floating bubbles instead of Windows toast notifications

## Lessons Learned

- **Mouse hooks are expensive** — a system-wide mouse hook for scroll-to-volume added ~0.5% idle CPU. Replaced with a tray notification callback for middle-click mode toggle (zero overhead).
- **Sync timer intervals matter** — reducing periodic mute-state sync from 3s to 5s cuts background wakeups without perceptible lag.
- **AHK v2 global scoping** — nested functions need explicit `global` declarations or variables silently shadow to local scope. Hit this with LoadConfig/SaveConfig and several other functions.
- **COM error handling** — always check HRESULT from GetMute/SetMute COM calls. Stale device handles return incorrect state silently.
- **Graceful degradation** — start in degraded mode (tray-only) if no mic is connected; auto-recover via periodic device check.
- **ToolTip over TrayTip** — Windows toast notifications are intrusive and unreliable. Floating ToolTip with SetTimer auto-dismiss is cleaner.
- **Embedded icons via @Ahk2Exe-AddResource** — eliminates dependency on external .ico files in compiled .exe. Fallback chain: custom INI path -> .ico on disk -> embedded PE resource -> Windows built-in.
- **AHK Hotkey GUI control** — cannot represent bare keys like `\` or `]`; returns empty on `.Value` read. Preserve old binding when control returns empty.
- **Production audit value** — post-release deep audit (v1.8.1) found 6 additional issues: wrong default hotkey text in Help window, OSD duration initial value mismatch, incorrect OSD position description, filename casing error in FINAL_REPORT, and stale multi-monitor claims in README and CHANGELOG. Always run a production audit before finalizing.
- **Documentation/code drift** — OSD position was described as "top-right corner" and "multi-monitor aware" in docs but actual code uses primary monitor only (Shell_TrayWnd). User-facing text audits catch this class of bug that code review misses.
